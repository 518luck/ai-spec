欢迎使用 GitLens。GitLens 极大地增强了 VS Code 内置的 Git 功能。无论你是经验丰富的 Git 老手，还是刚刚起步的新人，GitLens 都能让你更轻松、安全且高效地发挥 Git 的威力。

结合 Git 的强大功能，GitLens 提供了丰富的行内 Blame（追溯）注解和悬停提示。GitLens 会在当前代码行的末尾添加注解，显示作者姓名、相对时间戳和提交信息。将鼠标悬停在注解上，还能查看提交 ID、快捷操作、提交日期以及完整的提交信息等更多细节。
此外，GitLens 还在底部的状态栏显示 Blame 详情，你同样可以悬停或点击它来获取更多信息和操作选项。

点击编辑器工具栏上的对应图标，你可以开启“文件 Blame（File Blame）”视图。文件 Blame 会在每一行的开头提供详尽的注解，包括提交作者、信息、日期以及带颜色编码的热力图。

与行内 Blame 类似，文件 Blame 同样支持悬停提示，让你在探索历史变更时获得更多上下文。想要关闭文件 Blame，只需再次点击该图标或按下 Esc 键即可。此外，你还可以点击编辑器工具栏上的箭头图标，直接打开并查看上一次提交的变更。

另外，如果你按住 Alt 键（Mac 上是 Option 键）并点击，就能直接跳转并打开特定提交的变更。这里同样支持行内 Blame 和悬停提示，让浏览历史记录变得既简单又强大。除了编辑器顶部提供文件 Blame 的详细信息外，每个代码块也都可以通过 Git CodeLens 来查看相关信息。

点击“最近更改（recent change）”的 CodeLens，可以查看关于最新提交的更多信息，例如提交信息、提交 ID，或是该次提交中所有被修改文件的列表。类似地，点击“作者（authors）” CodeLens 会切换出之前介绍过的文件 Blame 视图；和之前一样，按 Esc 键即可退出。你还可以通过侧边栏的 GitLens 视图深入洞察你的代码仓库并执行相关操作。

点击活动栏（Activity Bar）中的“源代码管理”图标即可浏览侧边栏视图。在这里，你可以访问提交记录、历史、分支、远程仓库、暂存（Stashes）等众多内容。利用右键上下文菜单还能发掘更多选项。比如，只需在任意提交上右键点击，就能进行撤销提交（Revert）、重置（Reset）或创建分支等操作。

除了上述方式，许多操作选项也都可以通过命令面板（Command Palette）来调用。你可以使用快捷键 Cmd+Shift+P（Windows/Linux 下为 Ctrl+Shift+P）打开它，然后输入“GitLens”来访问各种 GitLens 命令，或是打开设置和帮助资源。GitLens 拥有极高的可定制性。现在让我们用快捷键调出命令面板，输入“GitLens: Settings”来打开它的设置页面。

在设置页面中，你可以根据个人喜好开启或关闭当前行 Blame、CodeLens、状态栏 Blame 等诸多功能。这是探索 GitLens 其他特色功能并打造专属体验的绝佳阵地。现在，你已经准备好使用这款 VS Code 中最受欢迎的 Git 扩展了。祝你编码愉快！
