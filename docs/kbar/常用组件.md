UI 组件：

- `KBarProvider`
  官方说明：应用最外层的上下文提供者，用来承载 actions、状态和快捷键能力。
  大白话：它就是 kbar 的总开关，不包这一层，后面的命令面板基本都跑不起来。

- `KBarPortal`
  官方说明：将命令面板内容渲染到根节点之外。
  大白话：它负责把弹窗“传送”到页面外层，避免被你当前页面布局卡住。

- `KBarPositioner`
  官方说明：用于定位命令面板内容，官方示例中通常用它来居中弹层。
  大白话：它管弹窗摆在哪里，一般就是把命令面板放到屏幕中间。

- `KBarAnimator`
  官方说明：负责命令面板显示、隐藏和高度变化时的动画。
  大白话：它让弹窗出现和收起时别那么生硬，看起来更像一个完整产品。

- `KBarSearch`
  官方说明：命令面板内置的搜索输入框组件。
  大白话：就是用户打字搜命令的那个输入框。

- `KBarResults`
  官方说明：用于渲染搜索结果的高性能虚拟列表组件。
  大白话：它负责把匹配到的命令一条条列出来，而且结果多的时候也尽量不卡。

Hooks：

- `useKBar`
  官方说明：用于读取 kbar 状态，并访问 query 等控制能力的 hook。
  大白话：你想手动开关命令面板、读当前状态、做一些自定义控制时，用它。

- `useMatches`
  官方说明：根据当前搜索词返回拍平后的匹配结果和分组信息。
  大白话：它会告诉你“现在搜出来了什么”，你拿这个结果去渲染列表就行。

- `useRegisterActions`
  官方说明：用于在运行时动态注册 actions 的 hook。
  大白话：有些命令不是一开始就写死的，而是组件加载后再加进去，这时用它。

工具和类型：

- `createAction`
  官方说明：用于创建或标准化 action 的辅助方法。
  大白话：它帮你把命令对象整理成 kbar 更容易处理的格式。

- `Priority`
  官方说明：用于定义 action 或分组优先级的工具。
  大白话：它决定哪些命令排前面，哪些排后面。

- `NO_GROUP`
  官方说明：kbar 内置的默认无分组标识。
  大白话：某条命令如果不想归到某个标题分组下面，就可以落到这个默认组里。

- `types` 里的各种类型
  官方说明：包含 action、provider、query、state 等 TypeScript 类型定义。
  大白话：写 TS 时用它们给参数和返回值标类型，少踩坑。

- `action` 相关导出
  官方说明：与 action 数据结构和实现相关的导出内容。
  大白话：这是更底层的一组东西，平时做普通命令面板 usually 不太需要直接碰。
