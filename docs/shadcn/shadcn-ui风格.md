# style风格

在 shadcn/ui 里，style 一般会影响这些东西：

- 组件的整体视觉气质
- 圆角、边框、阴影的倾向
- 按钮、输入框、弹层的默认外观
- 配色 token 的组织方式
- 某些组件的默认排版和间距
- 有时还会影响图标、图表、菜单等配套样式

## 各个风格

- Vega：经典 shadcn/ui 外观
- Nova：更紧凑，padding 和 margin 更少
- Maia：更柔和、更圆润，留白更大
- Lyra：更方、更锐利，适合搭配等宽/mono 风格
- Mira：紧凑型，适合信息密度高的界面
- Sera：2026 年 4 月新增，偏 editorial/排版感，强调下划线控件和大写标题

# 其他选项

## Base Color

基础主色调。
决定整套 UI 的基础色倾向，比如 neutral、stone、zinc 这类底色体系。

## Theme

主题色方案。
通常跟整体亮暗模式、语义色、组件 token 有关系。你可以理解成整体色彩气质配置。

## Heading

标题字体设置。
一般控制大标题、区块标题、卡片标题这类文字的字体风格。

## Font

正文字体设置。
一般控制正文、描述、按钮、输入框里的文字。

## Radius

圆角大小。
比如按钮、卡片、输入框、弹窗边角是更圆还是更直。

## Icon Library

这表示按钮图标、菜单图标、状态图标等，默认会偏向这套图标风格。

它影响的是：

- 图标线条粗细
- 转角风格
- 图标整体气质
- 是否更简洁、几何、圆润、技术感

## Menu

这里的 menu 不是“有没有菜单”，而是“菜单长什么样”。
通常会影响：

- 侧边栏菜单项的底色
- 菜单项选中时是实心、描边还是更轻的状态
- 菜单块的层次感
- 菜单整体是更硬还是更柔

## Menu Accent

意思是：菜单的强调方式。

也就是菜单项在：

- hover
- active
- selected
- focused

这些状态下，用什么方式突出。
