- TestSession : 测试会话
- Workspace : 工作区
- User : 用户

## 再列动作：

- read : 读
- create : 创建
- update : 更新
- delete : 删除
- publish : 发布
- favorite : 收藏
- manage : 管理
- share : 分享

# 第二步：先做 RBAC 骨架

先定义角色，不要太多：

- guest : 游客
- user : 普通用户
- creator : 内容创作者
- admin : 管理员

然后写一张最简单的“角色 -> 动作”表。比如：

- guest 可以 prompt.read
- creator 可以 prompt.create
- admin 可以 prompt.publish

# 第三步：再补 ABAC 条件

当你发现“光靠角色不够”时，再补属性规则。

例如：

- creator 不等于可以编辑所有 prompt
- 只能编辑 authorId === user.id 的 prompt
- published 的 prompt 也许作者不能直接删除
- private 的内容只有 owner 能看
