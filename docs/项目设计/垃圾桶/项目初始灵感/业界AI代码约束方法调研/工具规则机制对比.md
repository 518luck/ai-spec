# 主流 AI 编程工具的规则机制对比

> 对比 Claude Code、Cursor、VS Code Copilot、OpenCode 四大工具的规则配置能力。

---

## 一、规则文件

| 特性               | Claude Code             | Cursor               | VS Code Copilot                   | OpenCode                          |
| ------------------ | ----------------------- | -------------------- | --------------------------------- | --------------------------------- |
| 项目规则文件       | `CLAUDE.md`             | `.cursor/rules/*.md` | `.github/copilot-instructions.md` | `AGENTS.md`                       |
| 全局规则文件       | `~/.claude/CLAUDE.md`   | —                    | —                                 | `~/.config/opencode/AGENTS.md`    |
| 子目录级规则       | 支持                    | 支持                 | 支持 `*.instructions.md`          | 支持                              |
| 个人规则（不提交） | `CLAUDE.local.md`       | `.cursor/rules/`     | —                                 | —                                 |
| 外部文件引用       | `@path/to/file` 语法    | 手动引用             | 手动引用                          | `opencode.json` instructions 字段 |
| 远程 URL 引用      | 不支持                  | 不支持               | 不支持                            | 支持                              |
| 兼容性             | OpenCode 兼容 CLAUDE.md | 独立体系             | 独立体系                          | 兼容 CLAUDE.md                    |

---

## 二、高级约束机制

### 2.1 Skills / 知识包

| 工具            | 机制         | 路径                          | 特点                        |
| --------------- | ------------ | ----------------------------- | --------------------------- |
| Claude Code     | Skills       | `.claude/skills/*/SKILL.md`   | 按需加载，支持元数据和参数  |
| OpenCode        | Agent Skills | `.opencode/skills/`           | 类似 Claude Code 的 Skills  |
| Cursor          | —            | —                             | 通过 rules 文件实现类似效果 |
| VS Code Copilot | Prompt Files | `.github/prompts/*.prompt.md` | 可复用的任务模板            |

### 2.2 自定义子代理

| 工具            | 机制                  | 特点                               |
| --------------- | --------------------- | ---------------------------------- |
| Claude Code     | `.claude/agents/*.md` | 独立上下文、独立工具权限、可选模型 |
| OpenCode        | Agents 配置           | 支持自定义代理                     |
| Cursor          | —                     | 不支持                             |
| VS Code Copilot | Custom Agents         | `.github/agents/*.agent.md`        |

### 2.3 Hooks（生命周期钩子）

| 工具            | 机制                          | 用途                                  |
| --------------- | ----------------------------- | ------------------------------------- |
| Claude Code     | `.claude/settings.json` hooks | 文件编辑后自动 lint、阻止写入特定目录 |
| OpenCode        | Hooks 配置                    | 类似 Claude Code                      |
| Cursor          | —                             | 不支持                                |
| VS Code Copilot | Hooks                         | 文件编辑后自动格式化等                |

### 2.4 MCP 服务器（外部工具集成）

| 工具            | 支持情况 | 用途                      |
| --------------- | -------- | ------------------------- |
| Claude Code     | 完整支持 | 连接数据库、API、Figma 等 |
| OpenCode        | 完整支持 | 连接外部服务和数据源      |
| Cursor          | 完整支持 | 连接外部工具              |
| VS Code Copilot | 完整支持 | 连接外部工具和数据        |

## 三、权限与安全

| 工具            | 权限模式             | 说明                   |
| --------------- | -------------------- | ---------------------- |
| Claude Code     | Auto / 白名单 / 沙箱 | 安全分类器自动判断风险 |
| OpenCode        | 权限规则配置         | 支持工具级别的权限控制 |
| Cursor          | Auto / 手动确认      | 类似 Claude Code       |
| VS Code Copilot | 工具级别控制         | 可配置允许/禁止的操作  |

## 四、初始化与自动生成

| 工具            | 命令    | 行为                                     |
| --------------- | ------- | ---------------------------------------- |
| Claude Code     | `/init` | 扫描项目，生成 CLAUDE.md                 |
| OpenCode        | `/init` | 扫描项目，生成 AGENTS.md；已有文件则改进 |
| Cursor          | —       | 手动创建                                 |
| VS Code Copilot | `/init` | 生成 `.github/copilot-instructions.md`   |

---

## 五、总结

所有工具的核心思路一致：

```
规则文件（告诉 AI 怎么做）
+ Skills/Prompts（封装可复用的工作流）
+ Linter/Formatter（自动检测违规）
+ Hooks（强制执行）
+ 人工审查（最终兜底）
```

差异在于：

- **Claude Code** 和 **OpenCode** 的约束机制最完善（Skills + Agents + Hooks + MCP）
- **Cursor** 侧重 rules 文件和 MCP
- **VS Code Copilot** 侧重指令文件和自定义代理

选择哪个工具，规则文件的写法原则是相通的：**精简、明确、只写 AI 猜不到的东西**。
