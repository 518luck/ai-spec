# AI Spec — 产品设计文档 v1.0

## 1. 产品概述

### 1.1 产品名称

**AI Spec**（内部代号 `prompt-shelf`）

### 1.2 产品定位

面向 AI 从业者和开发者的 **AI 配置管理平台**。提供 AGENTS.md 片段、Prompts、Skills、Agents 四类核心素材的创建、管理、自由组合和团队协作能力，同时通过 MCP 协议实现与外部 AI 工具链的双向同步。

### 1.3 目标用户

| 用户类型      | 典型场景                                          |
| ------------- | ------------------------------------------------- |
| AI 应用开发者 | 管理 prompt 模板，编排 agent 配置，导出 AGENTS.md |
| Prompt 工程师 | 编写和测试 prompt，发布到社区获得反馈             |
| 团队负责人    | 搭建团队工作空间，统一管理共享的 AI 配置          |
| AI 工具用户   | 发现社区优质素材，一键导入到自己的工具链          |

### 1.4 核心价值主张

- **统一管理**：四种 AI 配置素材（snippet / prompt / skill / agent）在一个平台内管理
- **自由组合**：通过拖拽编排将素材组合成完整的 Agent 配置或 AGENTS.md 文件
- **团队协作**：工作空间支持多人实时协作编辑和权限管理
- **生态互通**：MCP 双向同步，与 Cursor、opencode 等 AI 工具无缝对接

## 2. 核心概念

### 2.1 Block（构建块）

平台中所有内容的最小管理单元，统一为 `Block` 实体。每个 Block 有且仅有一种类型：

| 类型           | 标识      | 说明                                           | 典型内容                               |
| -------------- | --------- | ---------------------------------------------- | -------------------------------------- |
| AGENTS.md 片段 | `snippet` | AGENTS.md 文件的组成部分，可拼装成完整配置文件 | 代码风格规则、项目结构说明、数据库约束 |
| Prompt         | `prompt`  | 提示词模板，支持变量占位符                     | "请将以下 {{language}} 代码重构为……"   |
| Skill          | `skill`   | AI 技能定义，描述一项具体能力                  | 代码审查技能、测试生成技能             |
| Agent          | `agent`   | 代理配置，由其他 Block 组合编排而成            | 完整的前端开发 Agent                   |

### 2.2 Category（分类）

Block 的分类维度，采用 **预定义 + 自定义** 的混合体系：

**预定义分类**（系统内置，不可删除）：

| 分类     | 适用类型 | 说明                         |
| -------- | -------- | ---------------------------- |
| 代码风格 | snippet  | 代码规范、命名规则、格式约定 |
| 项目结构 | snippet  | 目录组织、文件布局说明       |
| 数据库   | snippet  | ORM 规则、迁移约束           |
| 代码生成 | prompt   | 从描述生成代码的 prompt      |
| 文本处理 | prompt   | 翻译、摘要、改写类 prompt    |
| 分析审查 | prompt   | 代码审查、逻辑分析类 prompt  |
| 开发工具 | skill    | 面向开发场景的技能           |
| 写作工具 | skill    | 面向内容创作的技能           |
| 全栈开发 | agent    | 全栈开发场景的完整 Agent     |
| 专项助手 | agent    | 特定领域的 Agent             |

**自定义分类**：用户可在任意类型下创建自定义分类，支持树状层级（最深 2 级）。

### 2.3 Composition（组合关系）

Block 之间通过组合关系形成 DAG（有向无环图）：

- `snippet` → 可以组合进 `agent` 或导出为完整 AGENTS.md
- `prompt` → 可以转换为 `skill`，或组合进 `agent`
- `skill` → 可以组合进 `agent`
- `agent` → 可以引用 `snippet`、`prompt`、`skill`，也可以嵌套其他 `agent`

组合关系存储为 `BlockComposition` 表，记录父 Block 和子 Block 的引用及排序。

### 2.4 Space（工作空间）

团队协作的基本单元：

- 每个 Space 有独立的成员和权限
- Space 内的 Block 默认对成员共享
- 支持活动流，追踪成员的编辑和变更

### 2.5 MCP Connection（MCP 连接）

与外部 AI 工具链的双向同步通道：

- **拉取（Pull）**：从 MCP 服务获取 Block 到平台
- **推送（Push）**：将平台 Block 推送到 MCP 服务
- **双向（Bidirectional）**：自动检测变更并双向同步

- 保持原有功能不变
- 添加必要的类型注解

#### 拖拽编排交互流程

1. 用户在左侧素材面板浏览或搜索可用 Block
2. 将 Block 拖拽到中间编排画布
3. 画布中可调整 Block 的顺序（上移/下移/拖拽排序）
4. 右侧属性面板实时显示选中 Block 的属性和整体预览
5. 点击"导出"可将编排结果下载为文件

## 4. 用户流程

### 4.1 创建 Block 流程

```
用户进入对应类型管理页 → 点击"新建"按钮
  → 填写基本信息（名称、描述、分类、标签）
  → 编辑内容（类型专属编辑器）
  → 保存草稿 / 保存并发布
  → 返回列表页查看新建的 Block
```

### 4.2 组合 Agent 流程

```
用户进入 Agents 管理页 → 点击"新建 Agent"
  → 填写 Agent 基本信息
  → 进入编排视图
  → 从左侧素材面板搜索/浏览可用 Block
  → 拖拽 Block 到编排画布
  → 调整顺序和配置
  → 右侧实时预览最终配置
  → 保存编排方案
  → 导出为 AGENTS.md / JSON / 其他格式
```

### 4.3 工作空间协作流程

```
用户创建工作空间 → 邀请成员（输入邮箱，选择角色）
  → 成员接受邀请
  → 在空间内共享 Block 和编排方案
  → 多人协作编辑（实时同步）
  → 活动流追踪所有变更
```

### 4.4 MCP 同步流程

```
用户在设置中添加 MCP 连接 → 配置端点和认证信息
  → 选择同步方向（拉取/推送/双向）
  → 配置 Block 类型到 MCP 资源的映射规则
  → 保存配置
  → 手动触发同步 或 等待自动同步
  → 查看同步日志确认结果
```

### 4.5 发现与导入流程

```
用户进入发现广场 → 浏览/搜索公开内容
  → 找到感兴趣的 Block
  → 查看详情和评分
  → 点击"导入"复制到个人素材库
  → 在个人管理页中查看导入的 Block
  → 可进一步编辑和自定义
```

## 5. 功能优先级

### P0 — MVP 核心功能

- Block CRUD（创建、读取、更新、删除）
- 统一数据模型（Block + BlockComposition）
- 分类系统（预定义分类 + 分类树导航）
- Snippets 管理（AGENTS.md 片段编辑器）
- Prompts 管理（模板编辑器 + 变量识别）
- Skills 管理（技能定义编辑器）
- Agents 管理（基本信息 + 简单列表引用）
- 标签系统

### P1 — 核心体验

- Agent 拖拽编排画布
- Snippet 组合预览和导出下载
- Prompt 变量系统和测试运行

- 工作空间创建和成员邀请
- 空间内共享素材和编排
- 发现广场（浏览、搜索、分类）
- Block 导入（从发现广场到个人素材库）

- 工作空间实时协作编辑

- 收藏和评分系统
- MCP 自动同步和同步日志
- Block 依赖分析和引用追踪
- 批量操作（批量导入、批量分类）
- 高级搜索（全文搜索、组合筛选）

### P3 — 生态功能

- 评分和评论系统

- Block 使用统计
- 推荐算法

## 6. RBAC 权限模型

### 6.1 全局角色

| 角色      | 说明         | 典型权限                       |
| --------- | ------------ | ------------------------------ |
| `guest`   | 未登录用户   | 只能浏览发现广场的公开内容     |
| `user`    | 普通登录用户 | 管理个人素材，加入工作空间     |
| `creator` | 内容创作者   | 管理个人素材 + 发布到发现广场  |
| `owner`   | 管理员       | 所有权限 + 用户管理 + 内容审核 |

### 6.2 资源与权限

| 资源              | 读取权限     | 写入权限       |
| ----------------- | ------------ | -------------- |
| Block（所有类型） | guest 及以上 | user 及以上    |
| Category          | 所有登录用户 | creator 及以上 |
| Space             | 成员可见     | 按空间角色     |
| MCP Connection    | 仅本人       | 仅本人         |

### 6.3 空间内角色

| 角色   | 查看素材 | 编辑素材 | 管理成员 | 管理空间 |
| ------ | -------- | -------- | -------- | -------- |
| viewer | ✅       | ❌       | ❌       | ❌       |
| editor | ✅       | ✅       | ❌       | ❌       |
| admin  | ✅       | ✅       | ✅       | ✅       |

## 9. 数据模型概要

> 详细字段定义见 `data-model.md`（待编写）

### 核心实体

```
Block (构建块)
├── id, title, description, content
├── type: snippet | prompt | skill | agent
├── visibility: private | shared | public
├── status: draft | published
├── categoryId → Category
├── authorId → User
├── spaceId? → Space
├── tags[] → BlockTag
├── metadata (JSON, 按 type 不同有不同结构)
├── childBlocks[] → BlockComposition (作为父 Block)
└── parentBlocks[] → BlockComposition (作为子 Block)

Category (分类)
├── id, name, icon, color
├── parentId? → Category (支持树状层级)
├── blockType: snippet | prompt | skill | agent | all
├── isSystem: boolean (系统分类不可删除)
├── sort
└── userId? (null 表示系统分类)

BlockComposition (组合关系)
├── parentId → Block
├── childId → Block
├── sortOrder: Int
└── @@unique([parentId, childId])

BlockTag (标签)
├── id, name
└── @@unique([name])

Space (工作空间)
├── id, name, description, icon
├── createdBy → User
├── members[] → SpaceMember
└── blocks[] → Block

SpaceMember (空间成员)
├── userId → User
├── spaceId → Space
├── role: viewer | editor | admin
└── @@unique([userId, spaceId])

McpConnection (MCP 连接)
├── id, name, endpoint, authToken?
├── syncDirection: pull | push | bidirectional
├── syncConfig (JSON, 映射规则)
├── lastSyncedAt?
├── status: active | inactive | error
├── userId → User
└── syncLogs[] → McpSyncLog

McpSyncLog (同步日志)
├── id, connectionId → McpConnection
├── direction: pull | push
├── status: success | partial | failed
├── details (JSON)
└── createdAt
```

### 实体关系图

```
User ──1:N── Block
User ──1:N── Space (createdBy)
User ──N:N── Space (通过 SpaceMember)

Block ──N:1── Category
Block ──N:N── BlockTag
Block ──N:1── Space (可选)

Block ──N:N── Block (通过 BlockComposition, DAG)

Space ──1:N── SpaceMember
Space ──1:N── Block

User ──1:N── McpConnection
McpConnection ──1:N── McpSyncLog
```
