## §1. 核心需求（来自产品确认）

### 1.1 双表分离的理由

PromptRecord 和 PromptDraft 功能差异很大，不做单表 + type 区分：

| 维度       | PromptRecord（收录）           | PromptDraft（草稿）    |
| ---------- | ------------------------------ | ---------------------- |
| 定位       | 正式收录的提示词，以文字为主题 | 轻量暂存，快速记录灵感 |
| 版本历史   | ✅ 完整版本树                  | ❌ 无                  |
| 标签分类   | ✅ 有                          | ❌ 无                  |
| 团队联动   | ✅ workspaceId                 | ❌ 无                  |
| 公开广场   | ✅ visibility = public         | ❌ 无                  |
| 收藏       | ✅ 可被收藏                    | ❌ 无                  |
| 图片       | ✅ 预留（未来支持）            | ✅ 可能含图片          |
| 最近编辑人 | ✅ lastEditorId                | ❌ 无（仅 owner）      |

### 1.2 排序需求

UI 上需要三种排序维度：

| 排序 | 驱动字段 / 表                           |
| ---- | --------------------------------------- |
| 最近 | `PromptRecord.updatedAt`                |
| 常用 | `PromptRecord.useCount`                 |
| 收集 | `PromptFavorite` 关联表（用户收藏记录） |

### 1.3 共享需求

- **含团队**：发布到团队空间，团队成员可见可编辑。
- **含公开广场**：发布到公共发现区，所有人可浏览/收藏/fork。

### 2.1 PromptRecord（提示词收录 — 核心实体）

```prisma
// 提示词收录主表，承载完整内容、版本、标签与共享
model PromptRecord {
  id              String     @id @default(cuid())
  title           String
  description     String?
  content         String     @db.Text
  images          String[]   @default([])        // 图片 URL 列表，预留扩展

  ownerId         String                          // 创建者
  owner           User       @relation("PromptRecordOwner", fields: [ownerId], references: [id], onDelete: Cascade)
  lastEditorId    String?                         // 最近编辑人（团队协作场景）
  lastEditor      User?      @relation("PromptRecordEditor", fields: [lastEditorId], references: [id], onDelete: SetNull)

  visibility      Visibility @default(private)    // private=私有，public=公开广场
  workspaceId     String?                         // null=个人，有值=团队副本
  sourceRecordId  String?                         // fork 来源追溯（无 FK，允许悬空）

  useCount        Int        @default(0)          // 使用次数，驱动"常用"排序
  viewCount       Int        @default(0)          // 浏览次数

  createdAt       DateTime   @default(now())
  updatedAt       DateTime   @updatedAt

  versions        PromptRecordVersion[]
  tags            PromptRecordTag[]
  favoritedBy     PromptFavorite[]

  @@index([ownerId])
  @@index([workspaceId])
  @@index([visibility])
  @@index([updatedAt])                             // 支持"最近"排序
  @@schema("public")
}
```

**字段说明**：

| 字段             | 类型              | 说明                              |
| ---------------- | ----------------- | --------------------------------- |
| `id`             | String (cuid)     | 主键                              |
| `name`           | String            | 名称                              |
| `description`    | String?           | 描述（可选）                      |
| `content`        | String (@db.Text) | 提示词正文                        |
| `images`         | String[]          | 图片 URL 列表，当前预留           |
| `ownerId`        | String            | 创建者 ID（FK → auth.User）       |
| `lastEditorId`   | String?           | 最近编辑人 ID（团队协作场景）     |
| `visibility`     | Visibility        | private / public                  |
| `workspaceId`    | String?           | null=个人，有值=团队副本          |
| `sourceRecordId` | String?           | fork 源记录 ID（无 FK，允许悬空） |
| `useCount`       | Int               | 使用次数（驱动"常用"排序）        |
| `viewCount`      | Int               | 浏览次数                          |
| `createdAt`      | DateTime          | 创建时间                          |
| `updatedAt`      | DateTime          | 更新时间（驱动"最近"排序）        |

### 2.2 PromptRecordVersion（版本树）

```prisma
// 提示词版本快照，parentVersionId 自引用构成完整版本树
model PromptRecordVersion {
  id              String   @id @default(cuid())
  recordId        String
  record          PromptRecord @relation(fields: [recordId], references: [id], onDelete: Cascade)

  parentVersionId String?                          // null=根版本，自引用构成树
  parentVersion   PromptRecordVersion?  @relation("VersionTree", fields: [parentVersionId], references: [id], onDelete: SetNull)
  childVersions   PromptRecordVersion[] @relation("VersionTree")

  name            String                           // 该版本名称快照
  content         String   @db.Text                // 该版本内容快照
  editorId        String
  editor          User     @relation("VersionEditor", fields: [editorId], references: [id], onDelete: Cascade)

  versionNumber   Int                              // 版本序号（主线上递增）
  message         String?                          // 类似 commit message

  createdAt       DateTime @default(now())

  @@index([recordId])
  @@index([parentVersionId])
  @@schema("public")
}
```

**版本树结构示意**：

```
  v1 (root, parentVersionId = null)
 /  \
v2   v3 (branch from v1)
|    |
v4   v5
```

- `parentVersionId = null` → 根版本
- 每个 version 指向其父版本，多个子版本可共享同一父版本（分支）
- `PromptRecord.content` 始终存储当前 HEAD 内容，读取最新版本无需 join
- 从历史版本分支：创建新 version，`parentVersionId = 历史版本 id`，同时更新 `PromptRecord.content`

### 2.4 Tag + PromptRecordTag（标签 / 分类）

```prisma
// 全局标签定义，可被多个 PromptRecord 共用
model Tag {
  id        String   @id @default(cuid())
  name      String   @unique
  createdAt DateTime @default(now())

  records   PromptRecordTag[]

  @@schema("public")
}

// 标签与收录的多对多关联
model PromptRecordTag {
  recordId  String
  record    PromptRecord @relation(fields: [recordId], references: [id], onDelete: Cascade)
  tagId     String
  tag       Tag          @relation(fields: [tagId], references: [id], onDelete: Cascade)

  createdAt DateTime @default(now())

  @@id([recordId, tagId])
  @@index([tagId])
  @@schema("public")
}
```

**设计说明**：

- 标签 = 分类，不再单独建文件夹 / 层级目录。
- `Tag.name` 全局唯一，用户自由创建（去重 / 合并管理留给后续版本）。
- 删除标签时，`PromptRecordTag` 关联记录级联删除，PromptRecord 本身不受影响。

## §3. 关键设计决策

| 决策点                 | 方案                                                 | 理由                                                                                  |
| ---------------------- | ---------------------------------------------------- | ------------------------------------------------------------------------------------- |
| Draft vs Record 分表   | 两张独立表                                           | 功能差异大（Record 有版本/标签/团队，Draft 无），单表 + type 会产生大量 nullable 字段 |
| 版本树实现             | `parentVersionId` 自引用 + 内容快照                  | 支持完整分支；`PromptRecord.content` 存当前 HEAD，读最新内容不需 join                 |
| parentVersion 删除策略 | `SetNull`（非 Cascade）                              | 删父版本时子版本变根，不会连锁删除整条历史                                            |
| lastEditor 删除策略    | `SetNull`（非 Cascade）                              | 用户注销不应删除被编辑的 prompt，仅清空编辑人记录                                     |
| 标签 = 分类            | 全局 `Tag` + M:N                                     | 不再单独建目录 / 文件夹，标签满足分类需求                                             |
| 排序支持               | `useCount` + `updatedAt` + 收藏表                    | 分别驱动"常用"、"最近"、"收集"三种排序                                                |
| 团队 / 广场            | `workspaceId`(null=个人) + `visibility`(public=广场) | 两个正交维度：workspaceId 控制团队归属，visibility 控制公开可见                       |
| fork 溯源              | `sourceRecordId` 无 FK                               | 沿用 v1 Resource 设计，源被删后允许悬空                                               |
| 图片                   | `String[]` 数组字段                                  | 当前够用；未来如需图片元数据（排序 / 说明）可升级为独立表                             |

## §4. 三种共享状态

```
个人私有:   workspaceId = null,  visibility = private   → 仅 owner 可见
团队共享:   workspaceId = <id>,  visibility = private   → 团队成员可见（通过 WorkspaceMember）
公开广场:   workspaceId = null,  visibility = public    → 所有人可浏览/收藏/fork
```

### 4.1 可见性查询逻辑

| 用户类型       | 可见的 PromptRecord                                                       |
| -------------- | ------------------------------------------------------------------------- |
| 游客（未登录） | `visibility = public`                                                     |
| 已登录用户     | `ownerId = 自己` ∪ `workspaceId ∈ 自己加入的团队` ∪ `visibility = public` |
| 团队成员       | 同上（团队维度由 WorkspaceMember 决定）                                   |

### 4.2 Fork 流程（从公开广场）

1. 用户在广场发现一条 `visibility = public` 的 PromptRecord
2. 点击 fork → 创建新 `PromptRecord`：
   - `sourceRecordId = 原记录 id`
   - `workspaceId = null`（个人副本）
   - `visibility = private`
   - `content` = 原记录当前内容
3. 新记录独立演化，版本树从零开始（首版本 `parentVersionId = null`）
4. 原记录与新记录互不影响（复制分离语义）

## §5. 实体关系总览

```
auth.User (cuid)
  ├── 1:N → PromptRecord           (owner, onDelete: Cascade)
  ├── 1:N → PromptRecord           (lastEditor, onDelete: SetNull)
  ├── 1:N → PromptRecordVersion    (editor, onDelete: Cascade)
  ├── 1:N → PromptDraft            (owner, onDelete: Cascade)
  └── 1:N → PromptFavorite         (user, onDelete: Cascade)

public.PromptRecord
  ├── 1:N → PromptRecordVersion    (onDelete: Cascade)
  ├── M:N → Tag                    (via PromptRecordTag, onDelete: Cascade)
  └── M:N → User                   (via PromptFavorite, onDelete: Cascade)

public.PromptRecordVersion
  └── 1:N → PromptRecordVersion    (parentVersion, self-ref "VersionTree", onDelete: SetNull)

public.Tag
  └── M:N → PromptRecord           (via PromptRecordTag, onDelete: Cascade)
```
