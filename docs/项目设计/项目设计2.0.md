# AI Spec — 项目设计文档 v2.0

## 1. 产品定位

AI Spec 是一个面向个人和团队的 AI 开发规约管理平台，用于收集高频 Prompt、沉淀单条规约、维护 AGENTS.md / Skills / Agents / Plugins，并在工作空间中按项目共享、复制、协作和追踪变更。

### 1.1 核心目标

- 快速收集和复用高频 AI 对话 Prompt。
- 用规约库沉淀可复用的单条规则、流程、权限和输出格式。
- 独立管理 AGENTS.md、Skills、Agents、Plugins 四类 AI 规约资产。
- 在工作空间中按项目组织 AI 规约，并支持多人协作和权限控制。
- 通过发现广场收藏、搜索、导入、发布公共资产。
- 在设置中管理 MCP、密钥和个人信息。

### 1.2 信息架构

```txt
一级导航
├── Prompt
├── 规约库
├── AI 规约
│   ├── AGENTS.md
│   ├── Skills
│   ├── Agents
│   └── Plugins
├── 工作空间
│   ├── 项目
│   ├── 成员
│   └── 共享资产（导航分组标题）
│       ├── AGENTS.md
│       ├── Skills
│       ├── Agents
│       └── Plugins
├── 发现
└── 设置
```

### 1.3 概念边界

| 概念 | 定位 | 典型用途 |
| --- | --- | --- |
| Prompt | 高频快捷话术 | 复制、收藏、编辑、分类、标签、历史记录 |
| 规约库 | 单条可复用规约 | 插入 AGENTS.md、Skills、Agents |
| AGENTS.md | 项目或目录级 AI 前置知识和约束文档 | 让 AI 理解项目架构、目录范围和开发规范 |
| Skill | 流程化技能 | 定义触发条件、执行步骤、资源文件和输出要求 |
| Agent | 角色、命令、权限和自主性配置 | 管理可执行命令、工具权限、可用 Skills 和绑定文档 |
| Plugin | 可复用打包单元 | 打包 Skills、Agents、AGENTS.md，便于跨项目复用 |
| 工作空间 | 团队和项目协作边界 | 项目划分、共享资产、成员权限、编辑记录 |

## 2. Prompt

### 2.1 定位

Prompt 是简短、高频、经常需要复制给 AI 的对话提示词。它不是项目规约，也不是 Skill 的严格流程，而是轻量快捷内容。

### 2.2 核心功能

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 创建 Prompt | 添加标题、内容、标签，新添加的 Prompt 默认排在前面 | P0 |
| 编辑 Prompt | 修改内容并生成单条历史版本 | P0 |
| 删除 Prompt | 删除或移入回收站 | P0 |
| 搜索 Prompt | 按标题、内容、标签搜索 | P0 |
| 标签管理 | 支持多标签筛选 | P0 |
| 点击复制 | 卡片一键复制 Prompt 内容 | P0 |
| 使用频率 | 记录复制次数、最近复制时间，复制频繁的 Prompt 自动靠前 | P0 |
| 常用标记 | 用户可手动设置常用，常用 Prompt 优先展示 | P0 |
| 默认排序 | 综合新建时间、使用频率、常用标记决定展示顺序 | P0 |
| 历史记录 | 查看全局 Prompt 编辑记录 | P1 |
| 单条历史 | 查看某一条 Prompt 的修改历史 | P1 |
| 发布到发现 | 将 Prompt 发布到广场 | P2 |

### 2.3 页面结构

```txt
Prompt
├── 顶部搜索栏
├── 标签筛选
├── 常用筛选
├── 使用频率排序
├── Prompt 卡片墙
├── 详情预览
├── 大弹窗编辑
└── 历史记录面板
```

### 2.4 Prompt 历史记录

Prompt 被修改时保留版本：

```txt
PromptRevision
├── promptId
├── titleSnapshot
├── contentSnapshot
├── tagsSnapshot
├── editedBy
├── changeSummary
└── createdAt
```

## 3. 规约库

### 3.1 定位

规约库保存单条可复用规约。它不是 Prompt，也不是完整 AGENTS.md，而是可以被 AGENTS.md、Skills、Agents 引用的原子内容。

### 3.2 规约分类与标签

规约库使用两层组织方式：第一层是必选的大类型，用于稳定导航和归档；第二层是自由标签，用于搜索、筛选和细分场景。

| 字段 | 规则 | 示例 |
| --- | --- | --- |
| 大类型 | 必选，每条规约只能选择一个大类型 | 通用、语言风格、基础设施、代码结构 |
| 标签 | 可自由添加，可设置多个标签 | 前端、后端、ts、js、react、prisma |

### 3.3 核心功能

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 增删改查 | 创建、编辑、删除、查看规约 | P0 |
| 大类型管理 | 每条规约必选一个大类型，用于稳定导航和归档 | P0 |
| 标签管理 | 支持自由添加多个标签，用于筛选和搜索 | P0 |
| 搜索 | 按标题、内容、大类型、标签搜索 | P0 |
| 引用到 AGENTS.md | 插入到某个文档章节 | P0 |
| 引用到 Skill | 插入到触发条件、流程或输出要求 | P0 |
| 引用到 Agent | 插入到系统指令、权限或命令说明 | P0 |
| 历史记录 | 查看规约被修改的记录 | P1 |
| 来源追踪 | 显示被哪些资产引用 | P1 |

### 3.4 重要规则

规约库中的规约被引入到 AGENTS.md / Skills / Agents 时，应允许两种模式：

- **引用模式**：源规约修改后，被引用位置可提示更新。
- **复制模式**：引入时生成快照，源规约修改不影响已插入内容。

项目工作空间中的引入默认使用复制模式，避免源文件后续修改影响项目已确认的规约内容。

AI 规约是 AGENTS.md、Skills、Agents、Plugins 的集合入口。它们都用于约束 AI 如何理解项目、如何执行任务、有哪些技能、能运行哪些命令、具备怎样的自主性。

```txt
AI 规约
├── AGENTS.md
├── Skills
├── Agents
└── Plugins
```

## 5. AGENTS.md

### 5.1 定位

AGENTS.md 是项目级或目录级的 AI 前置知识文档，用来告诉 AI 项目架构、目录责任、开发命令、规则边界和禁止事项。

### 5.2 核心功能

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 项目架构树 | 展示项目文件结构和 AGENTS.md 位置 | P0 |
| 约束范围标注 | 标注每个 AGENTS.md 生效目录和覆盖范围 | P0 |
| 文档编辑 | 点击文档后在编辑栏中编辑 | P0 |
| 增删改查 | 创建、编辑、删除、查看 AGENTS.md | P0 |
| 分类和标签 | 按项目类型、技术栈、目录用途分类 | P0 |
| 搜索 | 搜索文档标题、路径、内容、标签 | P0 |
| 引入 Prompt | 从 Prompt 选择内容插入当前文档 | P1 |
| 引入规约 | 从规约库选择规则插入当前文档 | P0 |
| 历史记录 | 查看所有 AGENTS.md 的编辑记录 | P1 |
| 单文档历史 | 查看某个 AGENTS.md 的版本历史 | P1 |
| 导出 / 下载 | 下载单个或项目内所有 AGENTS.md | P1 |
| MCP 同步 | 同步到项目目录或外部工具 | P2 |

### 5.3 页面结构

```txt
AGENTS.md
├── 左侧：项目架构树
│   ├── /AGENTS.md
│   ├── /src/AGENTS.md
│   ├── /app/api/AGENTS.md
│   └── /docs/AGENTS.md
├── 中间：Markdown 编辑器
└── 右侧：范围说明 / 引入 Prompt / 引入规约 / 预览 / 历史
```

### 5.4 范围标注

每个 AGENTS.md 应标注：

```txt
路径：/src/AGENTS.md
约束范围：src/**
说明：前端业务代码、FSD 分层、React、UI 组件规则
所属项目：ai-spec
可见性：私有 / 空间共享 / 公开
```

| 分类和标签 | 按场景、技能类型、技术栈分类 | P0 |
| 历史记录 | 查看 Skill 修改历史 | P1 |
| 结构化编辑 | 编辑名称、描述、触发条件、流程、输出要求 | P0 |
| 引入 Prompt | 将 Prompt 插入说明、流程或输出模板 | P1 |

| 资源文件 | 上传、下载、编辑资源文件 | P1 |
| 引用关系 | 查看被哪些 Agents 或 Plugins 使用 | P1 |
| 导出 / 同步 | 导出 Claude-like Skill 或同步到外部工具 | P2 |

## 7. Agents

### 7.1 定位

Agent 是角色、命令、权限、工具和自主性配置。它主要描述 AI 能做什么、怎么做、能运行哪些命令、能使用哪些 Skills 和 AGENTS.md。

### 7.2 核心功能

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 增删改查 | 创建、编辑、删除、查看 Agent | P0 |
| 搜索 | 按名称、目标工具、标签、权限搜索 | P0 |
| 分类和标签 | 按用途、工具、项目、团队分类 | P0 |
| 历史记录 | 查看 Agent 修改历史 | P1 |
| 目标工具 | Claude / Cursor / opencode / 自定义 | P0 |
| 角色说明 | 定义 Agent 角色和职责 | P0 |
| 系统指令 | 配置 Agent 的主提示和约束 | P0 |
| 命令设置 | 配置 Agent 可使用的命令 | P0 |
| 可视化命令列表 | npm、go、终端命令分组展示 | P0 |
| 权限设置 | 工具权限、文件权限、命令权限 | P0 |
| 引入规约 | 从规约库插入权限、规则或输出格式 | P0 |
| 关联 Skills | 配置 Agent 可使用的 Skills | P0 |
| 绑定 AGENTS.md | 配置 Agent 使用的项目规则文档 | P1 |
| 导出 / 同步 | 导出到目标工具或 MCP 同步 | P2 |

### 7.3 命令配置

Agent 编辑器中需要有可视化命令列表：

```txt
命令列表
├── npm / pnpm 命令
│   ├── pnpm run typecheck

│   └── pnpm run test
├── Go 命令
│   ├── go test ./...
│   └── go vet ./...
└── 终端命令
    ├── git status --short
    ├── docker compose up
    └── 自定义命令
```

每条命令应支持：

```txt
名称
命令内容
描述
风险等级
是否默认允许
是否需要确认
适用项目
```

| 导入 / 导出 | 下载或上传 Plugin 包 | P2 |

Plugin

## 9. 工作空间

### 9.1 定位

工作空间是多人协作和项目划分的核心界面。它下设项目、成员，以及共享资产分组；共享资产分组本身只是导航提示语言，不是独立页面，下面分别进入 AGENTS.md、Skills、Agents、Plugins。

### 9.2 工作空间结构

```txt
工作空间
├── 项目
│   ├── 项目列表 / 项目卡片
│   └── 项目详情
│       ├── 项目结构树
│       ├── 项目 AGENTS.md
│       ├── 项目 Skills
│       ├── 项目 Agents
│       ├── 项目 Plugins
│       └── 项目编辑记录
├── 成员
└── 共享资产（导航分组标题）
    ├── AGENTS.md
    ├── Skills
    ├── Agents
    └── Plugins
```

### 9.3 项目详情

点击某个项目后，用户可以看到：

```txt
项目：ai-spec
├── 项目结构
│   ├── /AGENTS.md
│   ├── /src/AGENTS.md
│   ├── /app/api/AGENTS.md
│   └── /docs/AGENTS.md
├── 已附带 Skills
├── 已附带 Agents
├── 已附带 Plugins
├── 同步状态
└── 编辑记录
```

### 9.4 复制隔离规则

从个人资产或共享资产引入到项目后，项目内生成独立副本。

```txt
源 AGENTS.md / Skill / Agent / Plugin
        ↓ 引入到项目
项目内副本
```

重要规则：

- 修改项目内副本，不影响源文件。
- 修改源文件，不自动影响已经引入到项目的副本。
- 可以提供“检查源文件更新”能力，但必须由用户手动确认更新。
- 项目内编辑器拥有与原资产同等的编辑能力。

### 9.5 工作空间编辑来源

在工作空间项目中编辑 AGENTS.md、Skills、Agents、Plugins 时，引入来源应是：

```txt
共享资产
```

而不是个人资产。

### 9.6 成员权限

成员可以设置空间级、项目级、文件级权限。

| 权限层级 | 能力 |
| --- | --- |
| 空间级 | 管理成员、共享资产、空间设置 |
| 项目级 | 查看或编辑某个项目 |
| 文件级 | 对某个 AGENTS.md / Skill / Agent / Plugin 设置可编辑或不可编辑 |

基础角色：

```txt
owner    空间所有权限
admin    管理成员和共享资产
editor   编辑被授权项目和文件
viewer   只读查看
```

### 9.7 共享资产

共享资产不包含 Prompt。共享资产在导航上不是独立页面，而是一个分组标题，分组下分别管理 AGENTS.md、Skills、Agents、Plugins。

```txt
共享资产
├── AGENTS.md
├── Skills
├── Agents
└── Plugins
```

共享资产支持：

- 上传
- 下载
- 从个人资产上传到共享资产
- 从共享资产引入到项目
- 搜索、标签、分类
- 权限控制
- 历史记录

### 9.8 编辑记录

编辑记录相当于操作日志：

```txt
ActivityLog
├── workspaceId
├── projectId?
├── assetType
├── assetId
├── action
├── actorId
├── summary
├── beforeSnapshot?
├── afterSnapshot?
└── createdAt
```

## 10. 发现

### 10.1 定位

发现是公开资产广场，用于搜索、收藏、导入和发布内容。

### 10.2 二级分类

```txt
发现
├── AGENTS.md
├── Skills
├── Agents
├── Plugins
└── 规约
```

Prompt 也可以发布和收藏，但不作为发现的核心二级结构，避免冲淡 AI 规约资产的主线。可在搜索筛选中选择 Prompt。

### 10.3 核心功能

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 搜索 | 搜索公开资产 | P0 |
| 标签筛选 | 按标签筛选 | P0 |
| 收藏 | 收藏公开资产 | P0 |
| 导入 | 导入到个人资产或工作空间共享资产 | P0 |
| 发布 | 发布自己的资产到发现 | P1 |
| 查看详情 | 查看说明、版本、作者、下载量 | P1 |
| 评分评论 | 对资产评分和评论 | P3 |

### 10.4 导入规则

```txt
AGENTS.md → AI 规约 / AGENTS.md 或工作空间共享资产
Skill     → AI 规约 / Skills 或工作空间共享资产
Agent     → AI 规约 / Agents 或工作空间共享资产
Plugin    → AI 规约 / Plugins 或工作空间共享资产
规约      → 规约库或工作空间共享资产
Prompt    → Prompt
```

## 14. 设计原则

- Prompt 保持轻量，不和规约库混淆。
- 规约库保存单条可复用规约，不承担完整文档职责。
- AGENTS.md、Skills、Agents、Plugins 独立管理，不强行一次生成。
- 工作空间按项目组织资产，项目内资产与源资产隔离。
- 共享资产不包含 Prompt，避免团队项目资产与个人快捷话术混杂。
- 发现负责公共分发，设置负责个人、MCP 和密钥。
