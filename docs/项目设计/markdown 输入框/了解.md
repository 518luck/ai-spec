# 展示

| 库               | 阶段          | 干什么                                                               |
| ---------------- | ------------- | -------------------------------------------------------------------- |
| remark           | Markdown 阶段 | 核心引擎，把 Markdown 文本解析成 AST（抽象语法树）                   |
| remark-gfm       | Markdown 阶段 | 插件，扩展支持 GitHub 风格语法（表格、删除线、复选框）               |
| remark-math      | Markdown 阶段 | 插件，识别 $...$（行内公式）和 $$...$$（块级公式），标记为"数学节点" |
| rehype           | HTML 阶段     | 把 Markdown AST 转成 HTML AST，继续加工                              |
| rehype-katex     | HTML 阶段     | 插件，拿到数学节点，调用 KaTeX 渲染成公式 HTML                       |
| rehype-highlight | HTML 阶段     | 插件，给代码块加语法高亮颜色                                         |
| react-markdown   | 最终输出      | 把上面整个链路串起来，输出 React 组件，直接渲染到页面                |

```
Markdown 原文                    这是 react-markdown 的内部流程
    │
    ▼
───────── remark 阶段（Markdown AST）─────────
    │
    │  remark-gfm    → 让它认识表格、删除线、任务列表（GitHub 风格）
    │  remark-math   → 让它认识 $x^2$ 和 $$\int$$ 是公式，不是普通文本
    │
    ▼
───────── rehype 阶段（HTML AST）─────────────
    │
    │  rehype-katex     → 把公式节点交给 KaTeX 渲染成 HTML
    │  rehype-highlight → 给代码块加语法高亮（颜色）
    │
    ▼
最终 React 组件
```
