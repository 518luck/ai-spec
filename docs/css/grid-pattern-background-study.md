# `InteractiveGridPattern` 学习笔记

## 1. 组件是什么

这个组件本质上是一个**页面级交互背景组件**。

它的职责不是承载业务逻辑，而是作为页面最外层的背景容器，提供这些视觉效果：

- 铺满整个屏幕的深色背景
- 规则排列的网格
- 鼠标靠近时的邻近高亮
- 鼠标悬停时的单格强高亮
- 中心环境光
- 边缘暗角

它通常这样使用：

```tsx
<InteractiveGridPattern>
  <LoginForm />
</InteractiveGridPattern>
```

也就是说：

- 背景效果在下面
- 页面内容在上面
- 它相当于整个页面的背景外壳

---

## 2. 这个效果是不是纯 CSS

不是纯 CSS。

它是由这几部分一起完成的：

- React：负责状态、事件和渲染
- Tailwind CSS：负责布局和基础样式
- 行内样式：负责动态颜色、阴影、尺寸
- 浏览器原生 API：负责尺寸监听

核心原因是：这个效果依赖鼠标位置和距离计算，而这些不是纯 CSS 能完成的。

例如它会用到：

```ts
useState
useEffect
onMouseMove
ResizeObserver
Math.sqrt(...)
```

所以更准确地说，它是：

**CSS 负责展示，JavaScript 负责交互计算。**

---

## 3. 这个组件用了什么

### 3.1 React

主要用了：

- `useState`
- `useEffect`
- `useRef`
- `useCallback`

### 3.2 Tailwind CSS

例如：

```tsx
fixed inset-0 overflow-hidden bg-neutral-950
absolute inset-0
relative z-10
border
transition-all
```

这些类负责：

- 固定铺满屏幕
- 绝对定位背景层
- 把内容层抬到上面
- 提供基础边框和动画

### 3.3 `cn`

项目里还用了：

```ts
import { cn } from "@/shared/lib/utils";
```

它通常是 `clsx + tailwind-merge` 的封装，用来：

- 拼接 `className`
- 处理 Tailwind 冲突类

### 3.4 浏览器原生 API

例如：

- `ResizeObserver`
- 鼠标事件

这部分不是第三方库，而是浏览器本身提供的能力。

---

## 4. 分层结构怎么理解

这个组件最重要的不是公式，而是**先理解层级结构**。

可以把它看成这样：

```tsx
<div className="fixed inset-0 overflow-hidden bg-neutral-950">
  <div>网格层</div>
  <div>中心光晕层</div>
  <div>暗角层</div>
  <div className="relative z-10">内容层</div>
</div>
```

层级含义：

1. 最底层：深色背景
2. 中间层：网格格子
3. 中间装饰层：中心光晕
4. 中间装饰层：暗角
5. 最上层：`children`

也就是说，这个组件不是“背景图”，而是一个**带内容层的背景容器**。

---

## 5. 为什么不影响 `children`

因为背景层和内容层是分开的，而且内容层被放到了更高层级。

关键结构是：

```tsx
<div className="fixed inset-0 overflow-hidden bg-neutral-950">
  <div className="absolute inset-0">背景层</div>
  <div className="relative z-10 h-full w-full">{children}</div>
</div>
```

这里的逻辑是：

- 外层是固定定位，负责铺满整个页面
- 背景层用绝对定位，只负责视觉展示
- `children` 被放到 `z-10` 的内容层里，所以在背景之上

因此它不会把内容“挤开”，而是让内容“盖在背景上面”。

---

## 6. `z-index` 这里怎么理解

很多人容易误会：**没写 `z-index` 不是 `-1`。**

默认值是：

```css
z-index: auto;
```

可以这样理解：

- `auto`：默认层叠规则，不主动声明具体层级
- `0`：明确指定在第 0 层
- `10`：明确指定在更高层
- `-1`：明确放到更后面

在这个组件里通常是：

- 背景层：`z-index: auto`
- 内容层：`z-10`

所以内容层自然压在背景层上面。

### 不建议一开始用 `z-index: -1`

因为 `-1` 很容易带来这些问题：

- 跑到父元素背景后面
- 被父元素裁掉
- 层叠关系变复杂

做页面背景更稳的方式通常是：

- 父容器负责定位
- 背景层用 `absolute`
- 内容层用 `relative z-10`

---

## 7. 从 0 复刻时，正确的搭建顺序

不要一开始就写鼠标发光。

正确顺序是：

**先搭层，再搭格子，最后加交互。**

---

## 8. 推荐的实现步骤

### 8.1 第一步：先做最外层背景容器

目标是先得到一个能铺满页面的背景层。

基础结构：

```tsx
<div className="fixed inset-0 overflow-hidden bg-neutral-950">
  <div>网格层</div>
  <div>光晕层</div>
  <div>暗角层</div>
  <div className="relative z-10">内容层</div>
</div>
```

这一阶段只验证两件事：

- 背景能不能铺满整个屏幕
- `children` 能不能显示在最上面

如果这一步没搭对，后面都会乱。

### 8.2 第二步：先做静态网格，不做交互

先别管鼠标，先把格子铺满一屏。

要先解决：

- 一共有多少行
- 一共有多少列
- 每个格子的宽高是多少

核心思路：

- 拿到容器宽高
- 用 `cellSize` 计算 `rows` 和 `cols`
- 双层循环渲染格子

只要能看到完整静态网格，就算成功。

### 8.3 第三步：做自适应尺寸

静态网格出来后，再解决窗口变化问题。

需要这些东西：

- `ref` 获取容器
- `getBoundingClientRect()` 读取尺寸
- `ResizeObserver` 监听容器变化
- `useEffect` 中重新计算网格

目标是：

- 窗口变化时格子数量自动变化
- 边缘不留白

### 8.4 第四步：记录鼠标位置

这一步先不要写复杂样式，只先拿到鼠标坐标。

需要：

- `mousePos`
- `onMouseMove`
- `onMouseLeave`

重点不是发光，而是确认你拿到的是：

**鼠标相对容器左上角的坐标。**

### 8.5 第五步：做距离计算

这是整个组件最核心的逻辑。

每个格子都要计算：

- 格子中心点坐标
- 鼠标到格子中心点的距离
- 距离换算成一个 0 到 1 的强度值

核心公式：

```ts
const dx = mousePos.x - cellX;
const dy = mousePos.y - cellY;
const distance = Math.sqrt(dx * dx + dy * dy);
const factor = Math.max(0, 1 - distance / proximity);
```

含义：

- 距离越近，`factor` 越接近 `1`
- 距离越远，`factor` 越接近 `0`
- 超出范围后，直接变成 `0`

这一步先把 `factor` 映射成透明度就够了，不要急着加阴影。

### 8.6 第六步：做 hover 强高亮

“邻近高亮”做好后，再单独加“直接悬停高亮”。

这一步单独维护：

- `hoveredCell`
- `onMouseEnter`
- `onMouseLeave`

逻辑上要分开：

- 悬停：强高亮
- 接近：弱高亮

不要把这两个状态混在一起。

### 8.7 第七步：最后做视觉润色

等逻辑都通了，再加这些：

- `boxShadow`
- 透明度衰减
- 中心光晕
- 暗角
- 过渡时间

这些都属于最后一层美化，不要最先做。

---

## 9. 可执行的开发计划

可以直接按这个顺序拆任务：

1. 背景容器和内容层
2. 静态网格渲染
3. 容器尺寸监听和网格重算
4. 鼠标坐标跟踪
5. 格子距离计算
6. 悬停高亮
7. 光晕、暗角、动画优化

---

## 10. 每一步的验收标准

- 第 1 步：背景铺满，内容不被挡住
- 第 2 步：格子完整铺满容器
- 第 3 步：窗口变化时网格不留白
- 第 4 步：能稳定拿到相对坐标
- 第 5 步：鼠标附近格子有渐变高亮
- 第 6 步：当前悬停格子明显更亮
- 第 7 步：整体视觉自然，不生硬

---

## 11. 最容易犯的错误

- 一开始就写完整视觉效果，调试会很痛苦
- 把网格数量计算和鼠标交互混在一起写
- 没先处理层级，导致 `children` 被挡住
- 直接用 `z-index: -1` 做背景，后面容易出问题
- 没有分阶段验证，每一步都堆在一起改

---

## 12. 最小可运行思路

如果要自己从头复刻，最 practical 的起点是：

- 先写一个 `fixed inset-0` 的背景容器
- 再写一个 `relative z-10` 的内容层
- 中间先放一个写死的静态网格

也就是说，不要一开始就做“完整发光背景”，而是先做：

**最小骨架版 -> 静态版 -> 动态版 -> 视觉增强版**

---

## 13. 一句话总结

这个组件的本质不是“炫技动画”，而是一个有明确层级和状态计算的页面背景容器。

复刻它时最重要的不是先写样式，而是先把这三件事搭稳：

- 背景层和内容层的结构
- 网格的生成方式
- 鼠标距离到高亮强度的映射逻辑
