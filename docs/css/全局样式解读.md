# :root

:root 是 CSS 里的一个伪类选择器，表示 文档的根元素。
在普通 HTML 页面里，根元素就是：

  <html>

所以这段：

```
:root {
--background: #ffffff;
--foreground: #171717;
}
```

本质上可以理解成：

> 把这两个 CSS 变量定义在整个页面最顶层的 <html> 上。

# 标签选择器

```css
body {
  background: var(--background);
  color: var(--foreground);
  font-family: Arial, Helvetica, sans-serif;
}
```

意思为

```
  选择器 {
    属性名: 值;
  }
```

# 媒体查询

## 第一层：@media

这是 CSS 的 at-rule（规则指令），专门用来写“满足某条件时才生效”的样式。

> 如果条件成立，就应用大括号里的 CSS。

## 格式

(媒体特性: 值)

```css
// 所以这句的意思就是：
// 如果用户当前偏好的颜色方案是 dark，那就应用里面的样式。
@media (prefers-color-scheme: dark) {
  :root {
    --background: #171717;
    --foreground: #f6f0e8;
  }
}
```

表示如果浏览器告诉我用户更喜欢深色模式，就把全局背景和文字变量改成深色版本

## 常见的媒体条件

### 1. 视口宽高

最常见，做响应式布局时一直在用。

@media (max-width: 768px) { ... }
@media (min-width: 1024px) { ... }
@media (max-height: 700px) { ... }

意思是：

- max-width：宽度不超过某值
- min-width：宽度不小于某值
- max-height / min-height：高度条件

常用于：

- 手机 / 平板 / 桌面适配
- 不同屏幕下切布局

———

### 2. 设备方向

判断横屏还是竖屏。

@media (orientation: portrait) { ... }
@media (orientation: landscape) { ... }

意思是：

- portrait：竖屏
- landscape：横屏

常用于：

- 移动端横竖屏适配
- 图表 / 全屏页面布局调整

———

### 3. 颜色主题偏好

就是你刚问的这个。

@media (prefers-color-scheme: dark) { ... }
@media (prefers-color-scheme: light) { ... }

常用于：

- 深色模式
- 浅色模式

———

### 4. 动画偏好

判断用户是否希望减少动画。

@media (prefers-reduced-motion: reduce) { ... }
@media (prefers-reduced-motion: no-preference) { ... }

意思是：

- reduce：用户希望减少动画
- no-preference：没有特别要求

常用于：

- 关闭复杂动画
- 降低晕动症影响
- 做无障碍优化

———

### 5. 对比度偏好

判断用户是否希望更高对比度。

@media (prefers-contrast: more) { ... }
@media (prefers-contrast: less) { ... }

常用于：

- 无障碍优化
- 提高可读性

不过这个没有前面几个常见。

———

### 6. 是否支持 hover

判断当前设备能不能悬停。

@media (hover: hover) { ... }
@media (hover: none) { ... }

意思是：

- hover：设备支持鼠标悬停
- none：设备不支持悬停，通常是触屏

常用于：

- 区分桌面和触屏设备
- hover 效果是否启用

———

### 7. 指针精度

判断输入设备是精细指针还是粗略指针。

@media (pointer: fine) { ... }
@media (pointer: coarse) { ... }

意思是：

- fine：精细指针，比如鼠标
- coarse：粗略指针，比如手指触摸

常用于：

- 按钮点击区域大小调整
- 交互优化

———

### 8. 打印

针对打印样式。

@media print { ... }
@media screen { ... }

常用于：

- 打印页面
- 隐藏不需要打印的 UI
- 调整纸张布局

———

# @supports

这是规则名字。
表示“做一个 CSS 能力支持检查”。

## 这和 @media 的区别

### @media

看的是环境条件
比如：

- 屏幕宽度
- 用户主题偏好
- 是否打印
- 是否横屏

例如：

@media (max-width: 768px) { ... }

意思是：
屏幕宽度小于等于 768px 时生效。

### @supports

看的是浏览器能力

例如：

@supports (display: grid) { ... }

意思是：
浏览器支持 Grid 布局时生效。
